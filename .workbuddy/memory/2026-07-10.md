## 第四轮改动 — 手机端自动下滑修复 + 动画交互优化

### 改动背景
用户反馈：手机端浏览时页面自动往下滑、不显示 Hero 首页；滚动加载动画体验不佳。

### 问题诊断
1. **自动下滑根因**：CSS `scroll-behavior:smooth` + 浏览器默认 `scrollRestoration:'auto'`，导致重新访问时浏览器恢复上次滚动位置并平滑滚动到该位置，形成可见的"自动下滑"动画
2. **Hero 内容溢出**：手机端 Hero 内容（头像+姓名+打字机+副标题+5关键词+4统计+简介+2链接+滚动指示器）总高度约 670px，超过小屏视口（375x667 可用约 547px），`100vh` 在 iOS 上包含浏览器工具栏区域进一步加剧
3. **动画问题**：`fade-in` 对所有 glass-card 一律添加 `opacity:0`（包括首屏可见元素造成闪烁）、位移过大（24px）、时长过长（0.6s）、nth-child 交错延迟作用域错误

### JS 修改（main.js）
- 顶部添加 `history.scrollRestoration='manual'` + `window.scrollTo(0,0)` 阻止浏览器恢复滚动位置
- 移除 CSS `scroll-behavior:smooth`，新增 `initSmoothScroll()` 仅对 `a[href^="#"]` 锚点链接启用平滑滚动（navOffset=70px）
- `initFadeInObserver` 重写：reduced-motion 直接跳过；初始视口内元素不添加 fade-in 类（避免首屏闪烁）；仅对 fold 下方元素添加 fade-in + observe
- rootMargin 调整为 `0px 0px -50px 0px`

### CSS 修改（style.css）
- `html` 移除 `scroll-behavior:smooth`（改为 JS 控制）
- `.hero` 使用 `100svh`（+ `100vh` fallback），padding 从 `120px 24px 60px` 减至 `100px 24px 48px`
- `.fade-in` 位移 24px→12px，时长 0.6s→0.35s，缓动改为 `cubic-bezier(0.25,0.1,0.25,1)`（ease-out）
- 移除 `.fade-in:nth-child()` 交错延迟（作用域错误）
- 768px 断点：hero padding `80px 20px 36px`、avatar 84px、隐藏 `.hero-scroll`、hero-links margin-bottom 归零、各间距压缩
- 480px 断点：hero padding `70px 16px 28px`、统计条改 2x2 grid（替代 flex-wrap）、链接保持横向排列（减小 padding 替代堆叠）

### 本地预览
- 服务器运行在 http://localhost:8771/（200 确认正常）
